## Context Summary

Built an invitation-only showcase site for REALITECH's portfolio of delivered XR/AR/VR
projects, from a Google Drive folder of ~33 demo videos. The site is a pure front-end on
GitHub Pages + a Cloudflare Worker access gate + private R2 video hosting. It is **live and
working end-to-end**: entry-code gate, intro-trimmed preview loops, gated full-video
playback, per-client access logging with Lark alerts, and a Book Demo form that feeds the
platform's CRM (cpn.realitech.vn).

## Key Decisions Made

| Decision | Rationale | Alternatives |
|----------|-----------|--------------|
| Whole-site entry gate (not per-video) | Client wanted private, lead-filtered showcase + know who views | Public previews + per-video gate |
| Full videos in R2 + Worker token | Real protection; can't bypass via URL | Client-side-only (insecure) |
| Per-client codes + access log + Lark | "biết rõ khách nào vào xem" | Single shared password only |
| Lark webhook for alerts | Native CF Email Sending returned 404 (beta not provisioned); user lives in Lark | Resend email; KV-log only |
| Book Demo → public `api.realitech.vn/leads` | Endpoint is `@Public()`, CORS already allows showcase origin | Worker proxy (unneeded) |
| Worker domain `gate.realitech.vn` | `api.realitech.vn` already existed (409 conflict) | api./showcase-api. |
| Full-align fonts to Be Vietnam Pro | User chose full platform consistency | Hybrid / tokens-only |
| Trim 5s intro from all clips | Most clips open with old partner-logo intro | Per-clip manual trimming |
